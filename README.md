# Paint

Этот проект представляет из себя простейший графический редактор, работающий в браузере.
Приложение создано с использованием фреймворка [Blazor](https://docs.microsoft.com/en-us/aspnet/core/blazor/?view=aspnetcore-5.0) (server-side).
На данный момент в Paint реализованы всего два инструмента для рисования: прямая линия и карандаш.
Откройте решение `Paint.sln`, соберите и запустите проект. Разберитесь с тем, как с программной точки зрения
организованы инструменты для рисования.

### Задание

**Создайте систему плагинов, расширяющих функциональность редактора.**
На данный момент добавление новых инструментов требует внесения изменений в проект, перекомпиляции
и редеплоя всего приложения. Ваша задача - доработать проект таким образом, чтобы достаточно было
скопировать плагин в рабочую директорию и перезапустить приложение, чтобы расширение автоматически
применились. Плагины включают в себя новые инструменты и должны быть выполнены в виде отдельных сборок.
Использовать какие-либо готовые решения для организации системы плагинов запрещено.

1. Выделите базовую сборку для всех плагинов. Такая сборка должна содержать минимум кода - только то,
что необходимо для разработчика плагина.
2. Напишите не менее двух плагинов, каждый из которых должен находиться в отдельной сборке. Например,
вы можете создать инструменты для рисования эллипса и прямоугольника. **Поместите плагины в решение
`PaintPlugins.sln`.** Каждый плагин должен зависеть только от базовой сборки из п. 1.
3. Модифицируйте Paint. Приложение на старте должно находить все плагины, скопированные в его рабочую
директорию, и автоматически загружать инструменты из них. **Проект Paint не должен в явном виде зависеть
ни от одного плагина!** Однако он должен зависеть от базовой сборки из п. 1.
